How the model runs

For a certain set of input parameters (the amounts of various gases, an initial temperature profile, surface properties, the amount of incoming solar radiation), we use a radiative transfer scheme to calculate radiative heating rates. We add the change in temperature to the input temperature profile, producing a new temperature profile. If the new temperature profile is unstable to convection, a convective adjustment is applied in the troposphere. This changes the temperature profile again, giving us another new temperature profile. Then we calculate changes in other atmospheric properties, such as the water vapour concentration, which depends on the temperature. The new set of parameters is used as input to the radiative transfer scheme and the whole process is repeated many times. Eventually, the total change to the temperature profile becomes very small, and we say that the system has reached equilibrium.

Konrad is written in python and is available on our github page for anyone to download and use. The radiative transfer scheme used in konrad is RRTMG, the development of which started in the 1990s, and the scheme continues to be used in many weather prediction and climate models. RRTMG is written in fortran and made accessible through python by climt.
